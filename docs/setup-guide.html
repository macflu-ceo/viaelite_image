로그인 시스템 셋업 가이드

VIA ELITE 상품카드 생성기 · 구글 시트 연동 인증
전체 소요 시간: 약 15~20분 · 난이도: 비개발자도 가능 · 비용: 0원

목차

  1. 구글 시트 만들기 (3분)
  2. 시트 ID 복사 (1분)
  3. Apps Script 코드 등록 (3분)
  4. Apps Script 배포 → URL 받기 (5분)
  5. HTML 파일에 URL 붙여넣기 (2분)
  6. 테스트 로그인 (1분)
  7. 사용자 추가/제거/비밀번호 변경
  8. 문제 해결

1구글 시트 만들기

  1. sheets.google.com 접속 → 빈 스프레드시트 만들기
  2. 시트 제목을 "비아엘리떼 컨시어지 관리"로 변경 (좌상단 클릭)
  3. 화면 하단 시트 탭(기본값 "Sheet1")을 더블클릭하고 이름을 "컨시어지명단"으로 변경
  4. 1행에 다음 9개 컬럼 헤더를 그대로 입력 (순서 중요):
ABCDEFGHI
ID비밀번호이름활성레벨소속가입일마지막 로그인로그인 횟수

5. 2행부터 실제 컨시어지 정보를 입력합니다. 예시:

ID비밀번호이름활성레벨소속가입일마지막 로그인로그인 횟수
kim001vip2026김민지TRUE시니어강남지점2026-01-150
park002luxury88박지훈TRUE일반온라인팀2026-02-100
lee003elite2026이수민TRUE매니저청담편집샵2026-03-050
팁: "활성" 컬럼은 TRUE(활성) 또는 FALSE(비활성)로. 비활성으로 두면 로그인 차단됩니다. "마지막 로그인"과 "로그인 횟수"는 비워두세요 — 시스템이 자동으로 채웁니다.
중요 — 시트 공유 권한: 비밀번호가 평문으로 들어있으므로 시트 공유 권한을 본사 인원에게만 한정하세요. 컨시어지 본인에게는 공유하지 마세요. (우상단 "공유" → 본사 담당자만 추가)

2시트 ID 복사

브라우저 주소창의 URL에서 시트 ID를 찾아 복사합니다.

https://docs.google.com/spreadsheets/d/1A2B3C4D5E6F7G8H9I0J_여기가_시트_ID/edit#gid=0

위에서 굵게 표시된 부분(/d/와 /edit 사이)이 시트 ID입니다. 그 값을 메모장에 임시로 복사해두세요.

3Apps Script 코드 등록

  1. 방금 만든 시트에서 상단 메뉴 확장 프로그램 → Apps Script 클릭
  2. 새 탭으로 Apps Script 편집기가 열림. 기본 코드 function myFunction() {...}를 전부 선택해서 삭제
  3. server/viaelite_auth.gs 파일을 메모장 등에서 열어 내용 전체 복사
  4. Apps Script 편집기에 붙여넣기
  5. 코드 위쪽 const SHEET_ID = '여기에_본인_시트_ID_붙여넣기'; 부분의 따옴표 안 텍스트를 2단계에서 복사한 시트 ID로 교체
  6. 좌상단 프로젝트 이름 "제목 없는 프로젝트" 클릭 → "비아엘리떼 인증"으로 변경
  7. Ctrl+S (또는 Cmd+S)로 저장
연결 확인 (선택): 편집기 상단에서 함수 선택 드롭다운으로 testConnection 선택 → 실행 버튼 클릭. 최초 실행 시 권한 승인 요청 → 본인 구글 계정으로 승인. 하단 "실행 로그"에 헤더와 등록 사용자 수가 표시되면 성공.

4Apps Script 배포 → URL 받기

  1. 편집기 우측 상단 배포 (Deploy) 버튼 클릭 → 새 배포
  2. 좌측 톱니바퀴 ⚙️ 아이콘 클릭 → 웹 앱 선택
  3. 다음 정보 입력:
    • 설명: "비아엘리떼 인증 v1"
    • 다음 사용자 인증으로 실행: "나 (본인 이메일)"
    • 액세스 권한이 있는 사용자: "모든 사용자"
  4. 배포 버튼 클릭
  5. 권한 승인 화면이 뜨면 본인 구글 계정 선택 → "고급" → "프로젝트로 이동(안전하지 않음)" → "허용"
    "안전하지 않음" 경고는 정상입니다. 구글이 자체 검증을 안 거친 본인 스크립트라서 뜨는 표준 경고예요. 본인이 만든 스크립트이므로 안전합니다.
  6. 배포 완료 후 표시되는 웹 앱 URL을 복사 (https://script.google.com/macros/s/.../exec 형태)
이 URL이 핵심입니다. 다른 곳에 임시로 복사해두세요. 분실 시: Apps Script 편집기 → 배포 → 배포 관리에서 다시 확인 가능.

5HTML 파일에 URL 붙여넣기

  1. index.html을 메모장이나 텍스트 편집기로 열기 (브라우저 말고)
    맥: 파일 우클릭 → "다음으로 열기" → 텍스트 편집기. 윈도우: 파일 우클릭 → "연결 프로그램" → 메모장
  2. Ctrl+F(찾기)로 YOUR_APPS_SCRIPT_URL_HERE 검색
  3. 이 텍스트를 4단계 6번에서 복사한 웹 앱 URL로 교체. 결과는 아래처럼:
// 변경 전
const APPS_SCRIPT_URL = 'YOUR_APPS_SCRIPT_URL_HERE';

// 변경 후 (실제 URL 예시)
const APPS_SCRIPT_URL = 'https://script.google.com/macros/s/AKfycb.../exec';

4. 저장 (Ctrl+S)

6테스트 로그인

  1. index.html을 더블클릭해서 브라우저로 열기
  2. 로그인 화면이 뜨면 시트에 등록한 ID/비밀번호 입력 (예: kim001 / vip2026)
  3. "로그인" 클릭 → 카드 생성기 화면으로 이동하면 성공
  4. 구글 시트로 돌아가서 해당 사용자 행의 "마지막 로그인" 시각과 "로그인 횟수"가 자동으로 갱신됐는지 확인
  • 로그인 화면 우하단에 "✓ 시트 인증 활성화됨" 초록 배지가 보임
  • 잘못된 ID/비번 입력 시 "ID 또는 비밀번호가 일치하지 않습니다" 표시됨
  • 활성 컬럼이 FALSE인 사용자는 "비활성화된 계정입니다" 표시됨
  • 로그인 후 우상단에 사용자 이름·레벨·소속이 표시됨
  • 탭/창을 닫고 다시 열면 로그인 화면이 다시 나타남 (세션 만료 정상)

사용자 추가 / 제거 / 비밀번호 변경

새 컨시어지 추가

구글 시트에서 새 행에 정보 입력 → 시트가 자동 저장됨 → 즉시 로그인 가능. HTML 수정 불필요.

컨시어지 비활성화 (퇴사·자격박탈 등)

해당 행의 "활성" 컬럼을 TRUE에서 FALSE로 변경. 즉시 차단됨 (이미 로그인된 세션도 다음 로그인부터 차단).

비밀번호 변경

본사가 시트에서 직접 "비밀번호" 컬럼 값 수정. 본인이 바꾸는 기능은 현재 미지원 (필요하면 추후 추가 가능).

로그인 활동 확인

시트의 "마지막 로그인"과 "로그인 횟수" 컬럼이 자동으로 채워지므로, 누가 활발히 쓰는지 시트만 봐도 파악 가능.

문제 해결

"서버 연결 오류"가 뜸

  • 5단계에서 URL을 정확히 붙여넣었는지 확인 (앞뒤 따옴표 유지)
  • Apps Script 배포 시 "액세스 권한: 모든 사용자"로 설정했는지 확인
  • 배포를 한 번 더 새로 (배포 → 새 배포)

"비활성화된 계정입니다"가 뜸

  • 해당 사용자 행의 "활성" 컬럼이 TRUE인지 확인 (영문 대문자)

"시트를 찾을 수 없습니다"

  • 시트 하단 탭 이름이 정확히 "컨시어지명단"인지 확인 (공백·오타 주의)
  • 또는 Apps Script 코드의 SHEET_NAME 값을 본인 시트 탭 이름과 일치시킴

로그인 화면 하단에 "⚠️ 데모 모드" 배지가 뜸

  • HTML 파일의 APPS_SCRIPT_URL이 아직 YOUR_APPS_SCRIPT_URL_HERE인 상태. 5단계 다시 진행

이 시스템의 한계 (솔직한 설명)

  • 보안 수준: "비공식 내부 도구" 수준. 시트 공유 권한 관리 잘하면 충분히 안전. 단, 진짜 회원가입·결제·민감정보를 다루는 정식 서비스라면 별도 개발 필요.
  • 비밀번호: 평문 저장. 본사 시트 접근자만이 볼 수 있도록 권한 관리 필수.
  • 호출량: Apps Script는 일일 호출량 제한이 있음. 컨시어지 100명 안팎이면 무리 없음. 1,000명 이상으로 커지면 진짜 백엔드로 이전 권장.
  • 세션: 탭/창 닫으면 세션 만료(요청대로). 진짜 토큰 기반 인증 아님.
다음 단계: 컨시어지 베타 테스트로 5~10명 운영해보고, 반응 좋으면 (a) 외주 개발자에게 정식 회원시스템 의뢰 또는 (b) Claude Code로 직접 업그레이드 진행. 이 셋업은 그때까지의 임시 단계로 활용하기 좋아요.
